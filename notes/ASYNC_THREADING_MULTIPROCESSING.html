Asyncio vs Threading vs Multiprocessing

Asyncio

Concept: Single-threaded, cooperative multitasking using an event loop. Best for I/O-bound tasks.

When to Use: Many network calls, lightweight concurrency without threads.

Sample Code

import asyncio

async def fetch_data(n):
    print(f"Start task {n}")
    await asyncio.sleep(1)
    print(f"End task {n}")
    return n * 10

async def main():
    tasks = [fetch_data(i) for i in range(3)]
    results = await asyncio.gather(*tasks)
    print("Results:", results)

asyncio.run(main())

Key Parameters: asyncio.run(), asyncio.gather(), asyncio.sleep().

Threading

Concept: Multiple threads within one process, sharing memory. Limited by Python’s GIL.

When to Use: Parallel I/O tasks (downloads, DB queries).

Sample Code

import threading
import time

def worker(n):
    print(f"Thread {n} starting")
    time.sleep(1)
    print(f"Thread {n} done")

threads = []
for i in range(3):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

Key Parameters: threading.Thread(target, args), .start(), .join().

Multiprocessing

Concept: Multiple processes, each with its own interpreter and GIL. True parallelism on multiple cores.

When to Use: CPU-bound tasks (math, ML, image processing).

Sample Code

import multiprocessing
import time

def worker(n):
    print(f"Process {n} starting")
    time.sleep(1)
    print(f"Process {n} done")
    return n * n

if __name__ == "__main__":
    with multiprocessing.Pool(processes=3) as pool:
        results = pool.map(worker, [1, 2, 3])
    print("Results:", results)

Key Parameters: multiprocessing.Process(), multiprocessing.Pool(), .map().

Global Interpreter Lock (GIL)

Python threads are real OS threads, but the GIL ensures only one executes Python bytecode at a time. Threads are useful for I/O-bound tasks, but not CPU-bound tasks. Multiprocessing sidesteps the GIL entirely.

concurrent.futures

Provides a high-level API for threading and multiprocessing.

ThreadPoolExecutor

from concurrent.futures import ThreadPoolExecutor
import time

def io_task(n):
    print(f"Thread {n} start")
    time.sleep(1)
    print(f"Thread {n} end")
    return n * 10

with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(io_task, i) for i in range(3)]
    results = [f.result() for f in futures]

print("Results:", results)

Rule of Thumb

  • Use Asyncio for many lightweight I/O tasks.
  • Use ThreadPoolExecutor for blocking I/O tasks.
  • Use ProcessPoolExecutor for CPU-heavy tasks.

Side-by-Side Demo

Below is the same task — computing squares of numbers — implemented with Asyncio, ThreadPoolExecutor, and ProcessPoolExecutor.

Asyncio Version

import asyncio

async def compute_square(n):
    print(f"Async task {n} start")
    await asyncio.sleep(1)  # Simulates I/O wait
    print(f"Async task {n} end")
    return n * n

async def main():
    tasks = [compute_square(i) for i in range(5)]
    results = await asyncio.gather(*tasks)
    print("Results:", results)

asyncio.run(main())
