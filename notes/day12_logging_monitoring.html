C01

Why print() Fails in Production

▼
🌍 REAL WORLD — HOSPITAL PATIENT MONITORING Junior nurse (print()): "Patient's blood pressure is high" — no timestamp, no patient ID, no severity, disappears when shift ends.

Senior hospital system (structured logging): {"timestamp":"14:30Z","patient_id":"P-4521","event":"bp_reading","severity":"WARNING","value":"180/110","action":"notified_doctor"} — timestamped, searchable, permanent, alertable.

The specific problems in your agent system:

problems_with_print.txttext
print(f"📈 Price Agent: Fetching data for {ticker}...")
print(f"✅ ${current_price} | RSI: {rsi}")

Problems in production:
  1. No timestamp → WHEN did this analysis run?
  2. No request ID → WHICH of 500 concurrent requests is this for?
  3. No log level → Is this info, warning, or a critical error?
  4. No structured format → Can't query "all analyses where RSI > 70"
  5. No log file → If container restarts, logs are gone forever
  6. 500 concurrent requests → logs from all interleaved, unreadable

core/logger.pypython
import logging
import json
import sys
from datetime import datetime, timezone
from core.config import config

class StructuredFormatter(logging.Formatter):
    """
    Custom formatter that outputs JSON instead of plain text.
    Overrides format() — called automatically for every log message.
    """

    def format(self, record: logging.LogRecord) -> str:
        log_entry = {
            "timestamp": datetime.now(timezone.utc).isoformat(),
            # timezone.utc = consistent timestamps across timezones

            "level":    record.levelname,
            # "DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"

            "logger":   record.name,
            # which logger created this: "stock_agent.price_agent"

            "message":  record.getMessage(),
            "module":   record.module,
            "function": record.funcName,
            "line":     record.lineno,
            "version":  config.APP_VERSION
            # always tag logs with app version for debugging deployments
        }

        # Add any extra fields attached to this log record
        # This is how we add ticker, request_id, duration_ms, etc.
        if hasattr(record, "extra_fields"):
            log_entry.update(record.extra_fields)

        # Add traceback if this log includes an exception
        if record.exc_info:
            log_entry["exception"] = self.formatException(record.exc_info)

        return json.dumps(log_entry, ensure_ascii=False)


def get_logger(name: str) -> logging.Logger:
    """
    Creates a named logger. Call ONCE at module level — not inside functions.
    Usage: logger = get_logger("price_agent")
    """
    logger = logging.getLogger(f"stock_agent.{name}")
    # Hierarchical names let you control levels per part of system

    if not logger.handlers:
        # ── Console Handler ──────────────────────────────
        console_handler = logging.StreamHandler(sys.stdout)
        console_handler.setFormatter(StructuredFormatter())
        logger.addHandler(console_handler)

        # ── File Handler ──────────────────────────────────
        try:
            import os
            os.makedirs("logs", exist_ok=True)
            file_handler = logging.FileHandler("logs/stock_agent.log", encoding="utf-8")
            file_handler.setFormatter(StructuredFormatter())
            logger.addHandler(file_handler)
        except Exception:
            pass  # read-only filesystem → console logging still works

    logger.setLevel(logging.DEBUG)
    logger.propagate = False
    # propagate=False prevents logs bubbling to root logger (no duplicates)
    return logger


def log_with_context(logger: logging.Logger, level: str, message: str, **kwargs):
    """
    Helper to log with extra context fields cleanly.
    log_with_context(logger, "info", "msg", ticker="AAPL", rsi=58.4)
    """
    log_method = getattr(logger, level.lower())
    # getattr(logger, "info") → logger.info  (dynamic method lookup)
    log_method(message, extra={"extra_fields": kwargs})
core/metrics.pypython
import time
import json
from datetime import datetime, timezone
from memory.redis_client import redis_client

class AgentMetrics:
    """
    Tracks per-agent timing for every request.
    Stores in Redis so metrics persist and can be queried.
    """

    def __init__(self, request_id: str, ticker: str):
        self.request_id   = request_id
        self.ticker       = ticker
        self.start_time   = time.time()
        self.agent_timings = {}

    def start_agent(self, agent_name: str):
        """Call when an agent STARTS running"""
        self.agent_timings[agent_name] = {"start": time.time()}

    def end_agent(self, agent_name: str, success: bool = True, **extra_data):
        """Call when an agent FINISHES running"""
        if agent_name in self.agent_timings:
            start       = self.agent_timings[agent_name]["start"]
            duration_ms = round((time.time() - start) * 1000, 2)
            # × 1000 converts seconds → milliseconds
            self.agent_timings[agent_name].update({
                "duration_ms": duration_ms,
                "success": success,
                **extra_data  # e.g. cache_hit=True, risk_level="HIGH"
            })

    def save(self, final_recommendation: str = None):
        """Saves complete metrics record to Redis"""
        total_duration_ms = round((time.time() - self.start_time) * 1000, 2)

        metrics_record = {
            "request_id":           self.request_id,
            "ticker":               self.ticker,
            "timestamp":            datetime.now(timezone.utc).isoformat(),
            "total_duration_ms":    total_duration_ms,
            "agent_timings":        self.agent_timings,
            "final_recommendation": final_recommendation,
            "success": all(
                v.get("success", True)
                for v in self.agent_timings.values()
                if isinstance(v, dict) and "success" in v
            )
        }

        redis_client.set(f"metrics:{self.request_id}", json.dumps(metrics_record), ex=86400)

        # Add to sorted set — keeps N most recent by timestamp score
        redis_client.zadd("metrics:recent", {self.request_id: time.time()})
        redis_client.zremrangebyrank("metrics:recent", 0, -101)
        # zremrangebyrank(0, -101) = keep only the most recent 100

        return metrics_record


def get_recent_metrics(count: int = 10) -> list:
    """Returns metrics for most recent N requests. Used by /metrics endpoint."""
    recent_ids = redis_client.zrevrange("metrics:recent", 0, count - 1)
    # zrevrange = reverse order (newest first), positions 0 to count-1
    metrics = []
    for req_id in recent_ids:
        data = redis_client.get(f"metrics:{req_id}")
        if data:
            metrics.append(json.loads(data))
    return metrics

async def price_agent_node(state: StockAnalysisState) -> dict:
    ticker     = state["ticker"]
    request_id = state.get("request_id", str(uuid.uuid4()))
    # request_id flows through all agents via state
    # uuid4() = random unique ID like "a1b2c3d4-e5f6-..."

    log_with_context(logger, "info", "Price Agent started",
                     ticker=ticker, request_id=request_id, agent="price_agent")

    # ── Check cache ───────────────────────────────────────
    cached_data = get_cached("price", ticker)
    if cached_data:
        log_with_context(logger, "info", "Cache HIT — returning cached data",
                         ticker=ticker, request_id=request_id, cache_hit=True)
        return {**cached_data, "errors": []}

    # ── Fetch fresh data ──────────────────────────────────
    import time
    start = time.time()
    try:
        result      = await run_sync_in_thread(_fetch_price_sync, ticker)
        duration_ms = round((time.time() - start) * 1000, 2)

        log_with_context(logger, "info", "Price data fetched successfully",
                         ticker=ticker, request_id=request_id,
                         price=result["current_price"], rsi=result["rsi"],
                         duration_ms=duration_ms, cache_hit=False)

        set_cached("price", ticker, result)
        return {**result, "errors": []}

    except Exception as e:
        log_with_context(logger, "error", f"Price Agent failed: {str(e)}",
                         ticker=ticker, request_id=request_id, error=str(e))
        return {"current_price": None, "price_change_pct": None,
                "rsi": None, "errors": [f"Price Agent failed: {str(e)}"]}

INTERVIEW

Q&A — Day 12

Q1Why is print() insufficient for production logging and what does structured logging solve?▼
print() outputs unformatted text to stdout with no timestamp, no severity level, no request context, and no searchability. In production where thousands of requests run concurrently, you need to know WHEN something happened, FOR WHICH REQUEST, at what SEVERITY level, and you need to QUERY logs programmatically. Structured JSON logging gives every event a machine-readable format that can be stored in logging platforms like Datadog or Elasticsearch and queried instantly.
Q2Explain the log level hierarchy and how you'd configure it differently for dev vs production.▼
Log levels go DEBUG → INFO → WARNING → ERROR → CRITICAL. Setting a level filters out everything below it. In development I run at DEBUG to see every step of every agent. In production I run at WARNING because INFO-level logs for thousands of requests per minute would generate gigabytes of data and make actual problems hard to spot. Only WARNING and above represents something worth a human's attention in production — the normal INFO flow is just noise at scale.
Q3What is request tracing and why is it essential in a multi-agent system?▼
Request tracing assigns a unique request_id to each incoming request and threads it through every agent as part of the State. Every log line from every agent includes this ID. When a user reports a failed analysis, I query logs by request_id and instantly see the complete picture — which agent failed, how long each step took, what data was fetched — across all the agents that handled that specific request. Without it, logs from concurrent requests are interleaved and impossible to correlate.
Q4How does your metrics system help you catch performance degradation before users notice?▼
AgentMetrics tracks per-agent duration in milliseconds for every request and stores them in Redis with a 24-hour TTL. The /metrics endpoint surfaces recent timings. If news_agent duration suddenly spikes from 2 seconds to 8 seconds, it means Tavily's API is slowing down — visible in metrics before users start complaining. You can set alerts: "if any agent averages over 5 seconds for 10 consecutive requests, send a Slack notification." Reactive debugging turned into proactive monitoring.
Q5Why do you call get_logger() at module level instead of inside each function?▼
Creating a logger is not free — it involves looking up the logger registry, setting up handlers, configuring formatters. Calling it inside a function means this setup overhead happens on every single function call. At module level it runs once when the module is imported and the logger object is reused for every subsequent call from that module. It's also cleaner — the logger is a module-level resource that conceptually belongs to the module, not to individual function invocations.
Q6What would you add to make this observability system truly production-grade?▼
Several additions: first, centralized log aggregation — shipping logs to Datadog, CloudWatch, or a self-hosted ELK stack so logs from multiple instances are searchable in one place. Second, metrics dashboards in Grafana showing cache hit rate over time, average agent duration, error rates per ticker. Third, alerting — PagerDuty or Slack notifications when error rate exceeds 5% or when CRITICAL logs appear. Fourth, distributed tracing with OpenTelemetry so traces span across services if we split agents into microservices later.
