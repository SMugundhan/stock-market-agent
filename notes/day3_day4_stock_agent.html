What is Tool Calling?

▼
🌍 REAL WORLD — SURGEON IN OPERATION THEATRE The surgeon doesn't carry every instrument at once. They say "Scalpel please" — the nurse hands it. They use it. They request the next tool based on the result. That's Tool Calling: the LLM decides which function to invoke instead of guessing the answer.

C03

What is Sentiment Analysis?

▼
🌍 REAL WORLD — MARKET ANALYST READING HEADLINES You read 10 news articles about Tesla. Each one is positive, negative, or neutral. You aggregate them into one overall sentiment judgment. Sentiment analysis automates this — scoring the emotional and financial tone of text on a -1.0 to +1.0 scale.
scale.txttext
-1.0  ──────────────────────────────  +1.0
Very            Neutral           Very
Negative          0.0           Positive

-1.0 to -0.6  →  Very Negative  (sell signal)
-0.6 to -0.2  →  Negative
-0.2 to +0.2  →  Neutral
+0.2 to +0.6  →  Positive
+0.6 to +1.0  →  Very Positive  (buy signal)

Why LLM-based over rule-based (VADER)?
Financial news has nuance. "Apple's P/E ratio raises valuation concerns" — VADER sees no negative words and scores neutral. An LLM understands the financial implication is negative. LLM-based sentiment handles context, sarcasm, and domain-specific language far better.

Build the News Agent

agents/news_agent.pypython
from tavily import TavilyClient
# AI-optimized search client — returns clean JSON

from langchain_groq import ChatGroq
# Use LLM for sentiment — better than rule-based for financial news

from core.state import StockAnalysisState
from core.config import config

tavily_client = TavilyClient(api_key=config.TAVILY_API_KEY)
llm = ChatGroq(api_key=config.GROQ_API_KEY, model_name=config.MODEL_NAME)

def news_agent_node(state: StockAnalysisState) -> dict:
    ticker = state["ticker"]
    print(f"📰 News Agent: Fetching news for {ticker}...")

    try:
        # ── STEP 1: Fetch News ────────────────────────────────
        results = tavily_client.search(
            query=f"{ticker} stock news analysis today",
            max_results=5
            # 5 articles = enough signal, minimal token cost
        )
        articles = results.get("results", [])

        if not articles:
            return {"news_headlines": [], "sentiment_score": 0.0}

        headlines, contents = [], []
        for article in articles:
            title   = article.get("title", "")
            content = article.get("content", "")[:200]
            if title:
                headlines.append(title)
                contents.append(f"{title}: {content}")

        # ── STEP 2: Score Sentiment with LLaMA ───────────────
        news_text = "\n".join(contents)
        sentiment_prompt = f"""You are a financial news sentiment analyzer.
Analyze the sentiment of these news headlines about {ticker}:
{news_text}

Respond ONLY in this format:
SENTIMENT_SCORE: [decimal -1.0 to 1.0]
SENTIMENT_LABEL: [VERY_NEGATIVE/NEGATIVE/NEUTRAL/POSITIVE/VERY_POSITIVE]
SUMMARY: [one sentence]"""

        response = llm.invoke(sentiment_prompt)
        raw = response.content.strip()

        # ── STEP 3: Parse Response ───────────────────────────
        sentiment_score = 0.0
        for line in raw.split("\n"):
            if line.startswith("SENTIMENT_SCORE:"):
                sentiment_score = float(line.split(":")[1].strip())

        return {
            "news_headlines":  headlines,
            "sentiment_score": sentiment_score,
            "errors": []
        }

    except Exception as e:
        return {"news_headlines": [], "sentiment_score": 0.0,
                "errors": [f"News Agent failed: {str(e)}"]}

INTERVIEW

Q&A — Day 3

Q1 What is Tool Calling in AI agents? ▼
Tool Calling is the ability for an LLM to invoke external functions instead of just generating text. When the LLM recognizes it needs real-time data it can't have from training, it signals a tool call — the runtime executes the function, returns the result, and the LLM continues reasoning with actual data. This eliminates hallucination for factual queries.
Q2 Why did you choose Tavily over Google Search API? ▼
Tavily was built specifically for AI agents — it returns clean structured JSON with title, URL, and content snippet already extracted, so no HTML scraping is needed. Google Search API returns just URLs requiring additional scraping. Tavily also has a generous free tier and is optimized for low-latency LLM pipelines.
Q3 Why use LLM-based sentiment over rule-based tools like VADER? ▼
Financial news has nuanced language that rule-based tools miss. For example, "Apple's P/E ratio raises valuation concerns" — VADER sees no clearly negative words and scores it neutral, but an LLM understands the financial implication is negative. LLM-based sentiment handles context, sarcasm, and domain-specific language far better for financial analysis.
Q4 How does your News Agent handle the case where no news is found? ▼
The agent checks if the results list is empty after the Tavily call. If so, it returns empty headlines and a neutral sentiment score of 0.0 — representing no information rather than positive or negative. This prevents downstream agents from failing on None values and the Analyst Agent treats missing sentiment as a neutral signal.
Q5 What is the purpose of max_results=5 in the Tavily call? ▼
Limiting to 5 results balances coverage with token efficiency. More articles mean a larger prompt for the sentiment LLM call, which increases latency and API cost. 5 articles is enough to get a representative sentiment signal for most stocks. In production you could make this configurable based on the importance of the query.
Q6 How does adding the News Agent change the Analyst Agent's reasoning quality? ▼
Before Day 3 the Analyst only had price and RSI — purely technical signals. Adding sentiment gives a fundamental signal — what the market is thinking about the stock right now. A stock can have neutral RSI but very negative sentiment (sell signal) or oversold RSI but very positive sentiment (strong buy signal). The combination is far more powerful than either alone.

C02

What is Maximum Drawdown?

▼
🌍 REAL WORLD — ₹1,00,000 INVESTMENT Month 1: ₹1,00,000 → ₹1,20,000 (up 20%)
Month 2: ₹1,20,000 → ₹80,000 (crashes!)
Month 3: ₹80,000 → ₹90,000 (partial recovery)
Month 4: ₹90,000 → ₹1,10,000 (more recovery)

Worst point: ₹1,20,000 → ₹80,000 = -33.3% drawdown. That's what Maximum Drawdown captures.

Key insight: Volatility tells you how bumpy the daily ride is. Drawdown tells you the worst case scenario — the maximum pain an investor could have experienced. Fund managers say: "Never show me average returns. Show me the maximum drawdown. That's where investors panic and leave."

max_drawdown.pypython
cumulative   = (1 + daily_returns).cumprod()
# Running compound growth factor from all daily returns

rolling_max  = cumulative.cummax()
# At each point: "what was the peak value up to this date?"

drawdown_series = (cumulative - rolling_max) / rolling_max
# Negative values — how far below the peak are we?

max_drawdown = round(float(drawdown_series.min()) * 100, 2)
# e.g. -33.3 means fell 33.3% from the peak

# Formula: (Trough - Peak) / Peak × 100
# Example: (80 - 120) / 120 × 100 = -33.3%
C03

What is Beta?

▼
🌍 REAL WORLD — OCEAN WAVES AND BOATS The stock market = ocean waves. Individual stocks = boats.

Beta 1.0 → Boat moves exactly with the waves (stock tracks market)
Beta 1.5 → Boat amplifies waves (stock moves 1.5× the market — more risky)
Beta 0.5 → Boat barely moves (stock only reacts half as much — safer)
Beta -0.5 → Boat moves OPPOSITE (stock goes up when market goes down — rare)
beta_calculation.pypython
spy_returns = spy_hist["Close"].pct_change().dropna()
# SPY = S&P 500 ETF — industry-standard benchmark

min_length  = min(len(daily_returns), len(spy_returns))
stock_ret   = daily_returns.iloc[-min_length:].values
market_ret  = spy_returns.iloc[-min_length:].values
# Align lengths so comparison is fair

covariance       = np.cov(stock_ret, market_ret)[0][1]
# np.cov returns 2×2 matrix — [0][1] is the covariance element

market_variance  = np.var(market_ret)
# Beta formula: Cov(stock, market) / Var(market)

beta = round(covariance / market_variance, 2) if market_variance != 0 else 1.0

Beta vs Volatility: A stock can have low beta (moves little with the market) but high volatility (swings wildly on its own company news). They measure different things.

agents/risk_agent.pypython
import numpy as np
import yfinance as yf
from core.state import StockAnalysisState

def risk_agent_node(state: StockAnalysisState) -> dict:
    ticker = state["ticker"]
    print(f"⚠️  Risk Agent: Calculating risk for {ticker}...")

    try:
        stock = yf.Ticker(ticker)
        hist  = stock.history(period="6mo", interval="1d")
        # 6 months = enough to capture a full market cycle

        if hist.empty:
            return {"volatility": None, "risk_level": "HIGH",
                    "errors": [f"No historical data for {ticker}"]}

        closes       = hist["Close"]
        daily_returns = closes.pct_change().dropna()

        # ── Volatility ────────────────────────────────────────
        volatility_daily  = float(daily_returns.std())
        volatility_annual = round(volatility_daily * np.sqrt(252) * 100, 2)
        print(f"   📊 Annualized Volatility: {volatility_annual}%")

        # ── Max Drawdown ──────────────────────────────────────
        cumulative      = (1 + daily_returns).cumprod()
        rolling_max     = cumulative.cummax()
        drawdown_series = (cumulative - rolling_max) / rolling_max
        max_drawdown    = round(float(drawdown_series.min()) * 100, 2)
        print(f"   📉 Max Drawdown: {max_drawdown}%")

        # ── Beta vs SPY ───────────────────────────────────────
        spy         = yf.Ticker("SPY")
        spy_hist    = spy.history(period="6mo", interval="1d")
        spy_returns = spy_hist["Close"].pct_change().dropna()

        min_length  = min(len(daily_returns), len(spy_returns))
        stock_ret   = daily_returns.iloc[-min_length:].values
        market_ret  = spy_returns.iloc[-min_length:].values

        covariance      = np.cov(stock_ret, market_ret)[0][1]
        market_variance = np.var(market_ret)
        beta = round(covariance / market_variance, 2) if market_variance != 0 else 1.0
        print(f"   🎯 Beta: {beta}")

        # ── Risk Score (0-10) ─────────────────────────────────
        vol_score = 2.0 if volatility_annual < 15 else \
                    5.0 if volatility_annual < 25 else \
                    7.5 if volatility_annual < 40 else 10.0

        dd_score  = 2.0 if max_drawdown > -10 else \
                    5.0 if max_drawdown > -20 else \
                    7.5 if max_drawdown > -35 else 10.0

        beta_score = 2.0 if abs(beta) < 0.7 else \
                     4.0 if abs(beta) < 1.2 else \
                     7.0 if abs(beta) < 1.8 else 10.0

        risk_score = round((vol_score * 0.4) + (dd_score * 0.4) + (beta_score * 0.2), 2)

        risk_level = "LOW"    if risk_score <= 3.5 else \
                     "MEDIUM" if risk_score <= 6.5 else "HIGH"

        print(f"   ✅ Risk Score: {risk_score}/10 → {risk_level}")

        return {
            "volatility": volatility_annual,
            "risk_level": risk_level,
            "errors": []
        }

    except Exception as e:
        print(f"❌ Risk Agent Error: {e}")
        return {"volatility": None, "risk_level": "MEDIUM",
                "errors": [f"Risk Agent failed: {str(e)}"]}

INTERVIEW

Q&A — Day 4

Q1What is volatility and how do you calculate it?▼
Volatility measures how much a stock's price fluctuates. It's calculated as the standard deviation of daily returns, then annualized by multiplying by √252 (252 trading days per year). A stock with 25% annual volatility is considered medium risk — its price typically swings within a 25% band over a year. Higher volatility = higher potential gain but higher potential loss.
Q2What is Maximum Drawdown and why is it important?▼
Maximum Drawdown is the largest peak-to-trough percentage decline over a period. Formula: (Trough - Peak) / Peak × 100. It represents the worst case scenario — the maximum loss an investor could have experienced if they bought at the peak and held through the trough. Fund managers care more about drawdown than volatility because it's what causes investors to panic-sell.
Q3What is Beta and how does it differ from volatility?▼
Beta measures a stock's sensitivity to the overall market — calculated as Covariance(stock, market) / Variance(market). Beta of 1.5 means the stock moves 1.5× the market. Volatility measures the stock's own price swings regardless of the market. A stock can have low beta (moves little with market) but high volatility (swings wildly on its own news).
Q4How did you combine multiple risk metrics into a single score?▼
I created a weighted composite score — volatility gets 40%, max drawdown gets 40%, and beta gets 20%. Each metric is first normalized to a 0-10 sub-score based on standard thresholds (e.g. volatility <15% = score 2, >40% = score 10). The weighted sum gives a final 0-10 risk score. This mirrors how professional risk management frameworks like VaR combine multiple factors.
Q5Why did you use SPY as the benchmark for Beta calculation?▼
SPY is the most widely used S&P 500 ETF and represents the overall US stock market. Beta is always calculated relative to a benchmark — using SPY gives a standardized, industry-standard comparison. Every analyst and fund manager uses S&P 500 as the baseline market, so our Beta numbers are directly comparable to any professional risk report.
Q6What happens to your Analyst Agent's recommendation when Risk Level is HIGH?▼
The Analyst Agent's prompt explicitly includes risk level and provides guidance — HIGH risk + negative sentiment = strong sell signal, LOW risk + positive sentiment = strong buy signal. The LLM adjusts both its recommendation and confidence score accordingly. A HIGH risk stock with borderline positive sentiment might get HOLD instead of BUY, and with lower confidence — this is exactly how a real analyst would think.
