🧠 Concept 1

What is an AI Agent?

The Locked Room vs The Analyst
Why regular LLMs fall short for real-world tasks
🌍 Real World Scenario

Imagine you're a stock analyst at an investment firm. Your manager says: "Give me a full analysis of Apple stock by 5pm."

You don't just answer from memory. You open Bloomberg, read today's news, run risk calculations — then write the report. You used tools + reasoning + real data.

Regular LLMAI Agent
"Analyze Apple stock"
→ Answers from training data (months old)		"Analyze Apple stock"
→ Fetches live price, news, risk metrics
❌ No real-time data✅ Calls yFinance, Tavily, Risk tools
❌ Can hallucinate freely✅ Observes actual tool results
❌ Fixed one-shot response✅ Adapts based on what it finds
💡
Simple Definition: AI Agent = LLM + Tools + Memory + Reasoning Loop. The LLM is the brain — tools are its hands.
// What your Stock Market Agent does for "Analyze AAPL" Step 1 → I need price data → calls Price Tool → gets $189.3, RSI=58 Step 2 → I need latest news → calls Tavily Tool → gets 5 articles Step 3 → I need risk score → calls Risk Tool → volatility=0.34 Step 4 → I have enough data now → reasons over it Step 5 → Format the report → Report Agent → returns final JSON
🧠 Concept 2

🔄
The ReAct Pattern

Reasoning + Acting — The Brain of Every Agent
Used under the hood by LangGraph, CrewAI, AutoGen — every major agent framework
🌍 Real World Scenario — Sherlock Holmes

Holmes doesn't guess. He observes → thinks → acts → observes again in a loop until he can conclude. That's ReAct exactly.

Thought → "I need AAPL's current price" Action → call_tool(yfinance, ticker="AAPL") Observe → {price: 189.3, volume: 52M, change: +1.2%} Thought → "Now I need recent news about AAPL" Action → call_tool(tavily_search, query="Apple stock news today") Observe → [{title: "Apple hits record...", sentiment: "positive"}] Thought → "I have price + news. Now I can analyze." Answer → "AAPL at $189.3, up 1.2%, positive sentiment → HOLD"
⚠️
The agent loops until it decides it has enough information. It's not a fixed number of steps — it reasons about when to stop.

⛓️
Chain vs Agent

Assembly Line vs Surgeon
🌍 Real World Scenario

Chain = Assembly line. Every product goes through Machine 1 → 2 → 3. No flexibility. That's your RAG project.

Agent = Surgeon. Every operation is different. The surgeon picks tools based on what they find inside. That's this project.

Chain (Your RAG Project)Agent (This Project)
Fixed predetermined flowDynamic — LLM decides the flow
Input → Step1 → Step2 → OutputInput → Think → Maybe loop → Output
Always same orderOrder changes based on input
Simple, predictableFlexible, powerful
💡
Interview angle: "My RAG project was a chain — fixed flow. My stock agent is agentic — the LLM decides which tools to call, in what order, and when to stop. That's the evolution from RAG to Agents."

What is LangGraph?

Airport Operations System for AI
🌍 Real World Scenario

An airport has Air Traffic Control coordinating Gate Team, Fuel Team, Baggage Team — all sharing the same flight data. LangGraph is exactly this for your agents.

[User Request] ↓ [Orchestrator Agent] ← decides who to call ↙ ↘ [Price Agent] [News Agent] ↘ ↙ [Analyst Agent] ↓ [Report Agent] ↓ [Final Answer]

Each box = Node | Each arrow = Edge | Shared flight data = State

Temperature & Why LLM Answers Vary

Why doesn't the LLM always give the same answer?
This is a fundamental concept every AI Engineer must know
🌍 Real World Scenario

Ask 10 different stock analysts "Top 3 factors?" — each gives slightly different answers based on their experience and perspective. Temperature controls how different those answers are.

0.0
Deterministic
Always same answer
0.5
Balanced
Slight variation
1.0
Creative
More variation
2.0
Chaotic
Often nonsensical
python
# Same question, completely different answers based on prompt framing

# General top 3
llm.invoke("What are the top 3 factors to analyze before buying a stock?")

# Technical analysis focused
llm.invoke("What are the top 3 TECHNICAL INDICATORS before buying a stock?")

# Fundamental analysis focused
llm.invoke("What are the top 3 FUNDAMENTAL metrics a value investor checks?")

# In your project — you give REAL DATA, not vague questions
prompt = f"""You are a senior stock analyst.
Given: Price={price}, RSI={rsi}, Sentiment={sentiment}
Give a Buy/Hold/Sell recommendation with exactly 3 reasons."""
✅
In your project, you won't rely on the LLM's training knowledge. You'll feed it real data from your tools — so it reasons over facts, not guesses. This is why agents beat simple prompting.

Interview Questions & Answers

Q1 What is an AI Agent and how is it different from a regular LLM?
A regular LLM takes input and generates text from training data — it can't access real-time info or take actions. An AI Agent wraps an LLM with tools, memory, and a reasoning loop. It can call external APIs, observe results, and decide next steps dynamically — like a human analyst using tools to research before answering.
Q2 Explain the ReAct pattern with an example from your project.
ReAct = Reasoning + Acting. The agent loops: Thought → Action → Observation until it has enough to answer. In my stock agent: Thought → "need AAPL price" → Action → call yFinance → Observe → price=$189.3 → Thought → "now need news" → Action → call Tavily → and so on until it can generate the final recommendation.
Q3 What's the difference between a LangChain chain and an agent?
A chain has a fixed predetermined flow — like an assembly line, input always goes through the same steps in order. An agent is dynamic — the LLM decides which tools to call, in what order, and when to stop. My RAG project was a chain. This stock market project is an agent — that's the evolution.
Q4 Why did you choose LangGraph for multi-agent orchestration?
LangGraph models agents as a graph — nodes are agents/functions, edges are transitions. It supports shared state, conditional routing, and loops — essential for complex multi-agent coordination. Compared to CrewAI it gives production-level control; compared to AutoGen it's easier to deploy as a real API.
Q5 Why should API keys never be hardcoded in code?
Hardcoded keys pushed to GitHub become publicly visible — bots scrape them within minutes, leading to unauthorized usage and potentially large bills. Storing keys in a .env file + .gitignore ensures secrets never leave your local machine. This is professional standard practice — interviewers check for this.
Q6 Why does the LLM give different answers to the same question?
Because of Temperature — a parameter controlling randomness. At 0.0 it's deterministic (same answer always). At 1.0 it's creative and varied. Default is ~0.7. Also, the LLM doesn't have a "fixed answer list" — it predicts the most probable next token based on patterns from training data. Prompt engineering matters — "top 3 technical indicators" vs "top 3 fundamental metrics" gives completely different answers from the same model.
