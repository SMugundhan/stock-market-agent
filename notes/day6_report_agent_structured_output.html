C01

What is Structured Output?

▼
🌍 REAL WORLD — RADIOLOGIST READING AN X-RAY A radiologist doesn't write "looks like something in the left lung, seems okay, maybe some concern." They write a structured report with defined fields — PATIENT, FINDINGS, IMPRESSION, RECOMMENDATION. Anyone reading it instantly knows where to look. That's Structured Output — making LLM output follow a defined schema.

C02

What is Pydantic?

▼
🌍 REAL WORLD — CUSTOMS FORM AT AN AIRPORT Name, Passport No, DOB, Items value — each field has a required type and format. Write "abc" in the Items value field → form is INVALID. The form enforces the data contract. Pydantic does exactly this for Python data.
pydantic_model.pypython
from pydantic import BaseModel, validator
from typing import List

class StockReport(BaseModel):
    ticker:          str
    recommendation:  str
    confidence:      float
    risk_level:      str
    summary:         str
    key_factors:     List[str]
    price_target:    float | None

    @validator("recommendation")
    def recommendation_must_be_valid(cls, v):
        if v not in ["BUY", "HOLD", "SELL"]:
            raise ValueError("Must be BUY, HOLD, or SELL")
        return v

    @validator("confidence")
    def confidence_in_range(cls, v):
        if not 0.0 <= v <= 1.0:
            raise ValueError("Must be between 0 and 1")
        return v

Build the Report Agent

agents/report_agent.pypython
import json
# json.dumps(dict) → JSON string | json.loads(string) → dict
from datetime import datetime
from langchain_groq import ChatGroq
from core.state  import StockAnalysisState
from core.config import config

llm = ChatGroq(api_key=config.GROQ_API_KEY, model_name=config.MODEL_NAME)

def report_agent_node(state: StockAnalysisState) -> dict:
    ticker = state.get("ticker", "UNKNOWN")
    print(f"\n📋 Report Agent: Compiling report for {ticker}...")

    try:
        # ── STEP 1: Executive Summary via LLM ────────────────
        summary_prompt = f"""You are a professional financial report writer.
Write a concise 3-sentence executive summary:

Stock: {ticker}
Price: ${state.get('current_price', 'N/A')}
RSI: {state.get('rsi', 'N/A')}
Sentiment: {state.get('sentiment_score', 'N/A')}
Risk Level: {state.get('risk_level', 'N/A')}
Recommendation: {state.get('recommendation', 'N/A')}
Confidence: {state.get('confidence', 'N/A')}

Requirements: professional language, mention key
data points, end with recommendation + confidence.
Max 3 sentences, no bullet points."""

        summary_response  = llm.invoke(summary_prompt)
        executive_summary  = summary_response.content.strip()

        # ── STEP 2: Extract Key Factors programmatically ─────
        key_factors = []

        price_change = state.get("price_change_pct", 0) or 0
        if price_change > 1.5:
            key_factors.append(f"Strong upward momentum (+{price_change}%)")
        elif price_change < -1.5:
            key_factors.append(f"Downward pressure ({price_change}%)")
        else:
            key_factors.append(f"Price stable ({price_change}%)")

        rsi = state.get("rsi", 50) or 50
        if rsi > 70:
            key_factors.append(f"Overbought signal (RSI: {rsi})")
        elif rsi < 30:
            key_factors.append(f"Oversold signal (RSI: {rsi})")
        else:
            key_factors.append(f"RSI in neutral zone ({rsi})")

        sentiment = state.get("sentiment_score", 0) or 0
        if sentiment > 0.5:
            key_factors.append(f"Strong positive news sentiment ({sentiment})")
        elif sentiment < -0.3:
            key_factors.append(f"Negative news sentiment ({sentiment})")
        else:
            key_factors.append(f"Neutral market sentiment ({sentiment})")

        risk_level = state.get("risk_level", "MEDIUM") or "MEDIUM"
        volatility = state.get("volatility", 0) or 0
        key_factors.append(f"{risk_level} risk profile (volatility: {volatility}%)")

        # ── STEP 3: Build Structured Report Dict ─────────────
        report_dict = {
            "ticker":        ticker,
            "company_name":  ticker,
            "report_date":   datetime.now().strftime("%Y-%m-%d %H:%M"),
            "analysis_type": state.get("analysis_type", "full"),

            "market_data": {
                "current_price":    state.get("current_price"),
                "price_change_pct": state.get("price_change_pct"),
                "rsi":              state.get("rsi"),
            },

            "sentiment": {
                "score":     state.get("sentiment_score"),
                "headlines": state.get("news_headlines", [])[:3],
            },

            "risk": {
                "volatility_annual": state.get("volatility"),
                "risk_level":        state.get("risk_level"),
            },

            "verdict": {
                "recommendation": state.get("recommendation"),
                "confidence":     state.get("confidence"),
                "reasoning":      state.get("reasoning"),
                "key_factors":    key_factors,
            },

            "executive_summary": executive_summary,

            "disclaimer": (
                "This report is generated by an AI system for "
                "educational purposes only. It does not constitute "
                "financial advice. Always consult a qualified advisor."
            ),
            "generated_by": "Multi-Agent Stock Analysis System v1.0",
            "errors": state.get("errors", [])
        }

        # ── STEP 4: Convert dict → JSON string ───────────────
        final_report_json = json.dumps(report_dict, indent=2, ensure_ascii=False)

        print(f"✅ Report compiled — {report_dict['verdict']['recommendation']}")
        return {"final_report": final_report_json}

    except Exception as e:
        print(f"❌ Report Agent Error: {e}")
        # Never crash — return minimal fallback report
        fallback_report = json.dumps({
            "ticker": ticker,
            "report_date": datetime.now().strftime("%Y-%m-%d %H:%M"),
            "verdict": {
                "recommendation": state.get("recommendation", "UNAVAILABLE"),
                "confidence":     state.get("confidence", 0.0),
            },
            "error": f"Report generation failed: {str(e)}",
            "disclaimer": "This report is for educational purposes only."
        }, indent=2)
        return {"final_report": fallback_report}

from langgraph.graph import StateGraph, END
from core.state import StockAnalysisState

from agents.orchestrator  import orchestrator_node
from agents.price_agent   import price_agent_node
from agents.news_agent    import news_agent_node
from agents.risk_agent    import risk_agent_node
from agents.analyst_agent import analyst_agent_node
from agents.report_agent  import report_agent_node   # ← NEW
from agents.error_handler import error_handler_node
from agents.routing       import route_after_orchestrator, route_after_analyst

def build_graph():
    graph = StateGraph(StockAnalysisState)

    # ── All Nodes ──────────────────────────────────────────
    graph.add_node("orchestrator",  orchestrator_node)
    graph.add_node("price_agent",   price_agent_node)
    graph.add_node("news_agent",    news_agent_node)
    graph.add_node("risk_agent",    risk_agent_node)
    graph.add_node("analyst_agent", analyst_agent_node)
    graph.add_node("report_agent",  report_agent_node)  # ← NEW
    graph.add_node("error_handler", error_handler_node)

    graph.set_entry_point("orchestrator")

    graph.add_conditional_edges(
        "orchestrator", route_after_orchestrator,
        {"price_agent": "price_agent", "error_handler": "error_handler"}
    )

    graph.add_edge("price_agent", "news_agent")
    graph.add_edge("news_agent",  "risk_agent")
    graph.add_edge("risk_agent",  "analyst_agent")

    graph.add_conditional_edges(
        "analyst_agent", route_after_analyst,
        {
            "report_agent":  "report_agent",   # ← NOW routes to Report Agent
            "retry_analyst": "analyst_agent"
        }
    )

    graph.add_edge("report_agent",  END)   # ← NEW
    graph.add_edge("error_handler", END)

    return graph.compile()
run_day6.pypython
import json
from agents.graph import build_graph

def run_full_analysis(ticker: str):
    graph = build_graph()
    print(f"\n🚀 FULL PIPELINE: {ticker}")

    initial_state = {
        "ticker": ticker, "query_type": "full",
        "analysis_type": None, "should_fetch_news": None,
        "should_calculate_risk": None, "current_price": None,
        "price_change_pct": None, "rsi": None,
        "news_headlines": None, "sentiment_score": None,
        "volatility": None, "risk_level": None,
        "recommendation": None, "confidence": None,
        "reasoning": None, "final_report": None,
        "errors": [], "retry_count": 0
    }

    final_state = graph.invoke(initial_state)

    # json.loads() converts JSON string → Python dict
    try:
        report = json.loads(final_state.get("final_report", "{}"))
    except json.JSONDecodeError:
        print(f"\n📄 Report:\n{final_state.get('final_report')}")
        return

    print(f"Ticker: {report.get('ticker')}")
    print(f"Date:   {report.get('report_date')}")

    verdict = report.get("verdict", {})
    print(f"\n🎯 VERDICT")
    print(f"  Recommendation: {verdict.get('recommendation')}")
    print(f"  Confidence:     {verdict.get('confidence')}")
    for f in verdict.get("key_factors", []):
        print(f"    • {f}")

    print(f"\n📝 {report.get('executive_summary')}")

if __name__ == "__main__":
    run_full_analysis("AAPL")

INTERVIEW

Q&A — Day 6

Q1What is Structured Output and why is it important in production AI systems?▼
Structured output means the LLM returns data in a predefined schema — JSON with specific fields — rather than free-form text. In production this is critical because downstream systems need to programmatically access specific fields like recommendation, confidence, risk_level. Free-form text requires fragile text parsing that breaks when the LLM changes its phrasing. Structured output gives you a reliable contract between the LLM and the rest of your system.
Q2What is Pydantic and how does it help in agent systems?▼
Pydantic is Python's data validation library. You define a model class with typed fields and optional validators. When you create an instance, Pydantic automatically validates all values — wrong types, out-of-range values, or missing required fields raise ValidationError immediately. In agent systems this is essential because LLMs can return unexpected formats. Pydantic catches bad data at the boundary before it silently corrupts downstream processing.
Q3Why does your Report Agent generate a JSON report rather than plain text?▼
JSON enables programmatic consumption. The FastAPI layer can parse the JSON and return specific fields to the frontend. A mobile app can display just the recommendation badge. A database can store each field in its own column. A monitoring system can alert if confidence drops below 0.5. Plain text can't be consumed this way without fragile parsing. JSON is the industry standard for API responses precisely because it gives consumers a reliable data contract.
Q4How does your Report Agent handle partial data when an upstream agent fails?▼
Every field in the report uses state.get() with a default value — never direct dictionary access. The report agent always generates a report with whatever data is available. If the News Agent failed, sentiment fields show None but the rest of the report is intact. There's also a try/except with a fallback minimal report so even if the Report Agent itself crashes, the pipeline returns something usable rather than a raw Python exception.
Q5Walk me through the complete data flow from user input to final report.▼
User provides ticker and query type. Orchestrator validates and sets analysis flags. Price Agent fetches live price, change, RSI via yFinance. News Agent fetches headlines via Tavily, scores sentiment via LLaMA. Risk Agent calculates volatility, drawdown, beta from 6 months of historical data. Analyst Agent receives all this data and generates a BUY/HOLD/SELL recommendation with reasoning. Report Agent compiles everything into structured JSON with an LLM-written executive summary. Each step writes to State — the shared dictionary that flows through every node.
Q6What would you add to this report in a production system?▼
Several things: first, a price target based on technical analysis models. Second, historical recommendation accuracy tracking — how often did our BUY recommendations actually go up? Third, peer comparison — AAPL vs sector average. Fourth, earnings calendar integration — is there an earnings announcement coming that could invalidate the technical analysis? Fifth, portfolio fit analysis — given the user's existing holdings, does this recommendation make sense? These additions would move from a standalone analysis tool to a proper investment assistant.
